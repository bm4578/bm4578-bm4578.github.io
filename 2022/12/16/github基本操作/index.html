github基本操作

初始化本地仓库为GitHub仓库

  1. 将这个文件夹变成Git可管理的仓库。
  2. 将这个目录下的所有文件添加到仓库
  3. 提交说明
  4. 将本地仓库与GitHub上创建好的目标远程仓库进行关联。
  5. 创建主分支
  6. 将主分支提交到远程
  7. 查看远程主分支

提交本地文件到GitHub

  1. 查看当前分支情况，*标识的是你当前所在的分支。
  2. 将这个目录下的所有文件添加到仓库
  3. 提交说明
  4. 提交到远程分支

创建分支

  1. 查看当前分支情况，*标识的是你当前所在的分支。
    • 查看远程分支
    • 查看所有分支
  2. 新建一个分支test
  3. 切换到新建的分支test
    • 创建加切换分支
  4. 将新建分支test推送到GitHub上

合并分支

  1. 查看当前分支情况，*标识的是你当前所在的分支。
  2. 切换分支到test（需要合并的分支）
  3. 提交这个目录下的所有文件
  4. 提交说明信息
  5. 提交到test分支
  6. 再把工作从test分支下切换到main下
  7. 重新拉取主分支项目（防止别人已经提交了文件，此时你提交会覆盖别人提交的内容）
  8. 将test分支合并到main上

产生冲突

  1. 查看产生冲突文件
  2. 查看分支中内容提交的状态
  3. 把修改的内容提交到主分支上

若提交出现问题

  1. 撤销合并恢复到以前状态
  2. 撤销远程分支

删除分支

  1. 删除本地分支
  2. 删除远程分支
    PS: 删除主分支。思路：创建一个新分支，然后去官网设置新分支为主分支，然后再删除远程原主分支。

重命名分支

  1. 重命名本地分支
  2. 删除远程对应的旧分支
  3. 将本地的新分支推送到远程
